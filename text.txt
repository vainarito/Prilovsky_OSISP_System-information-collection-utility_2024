Ниже приведена информация выводимая при нажатии на кнопку
«Информация о работающих процессах».

** USER: Информация об имени пользователя может быть использована
для аудита и контроля безопасности. Пользователь может проверять, какие
процессы выполняются от имени конкретного пользователя, чтобы убедиться,
что нет несанкционированных или вредоносных действий. Также, эта
информация может быть использована для определения принадлежности
процессов к определенным пользователям и для анализа использования
ресурсов в рамках конкретного пользователя.

** PID: Идентификатор процесса может использоваться для управления
процессами или взаимодействия с ними. Пользователь может отправлять
сигналы процессу, используя его идентификатор, что позволяет
контролировать его поведение или завершить его работу. Также, PID может
использоваться для мониторинга процессов и их активности, например, для
определения процессов, использующих много ресурсов или вызывающих
проблемы.

** TIME: Значение времени выполнения процесса может быть
использовано для профилирования и оптимизации процессов. Пользователь
может анализировать время выполнения процессов, чтобы определить, какие
процессы занимают большую часть процессорного времени и могут быть
узкими местами в системе. Это может помочь в оптимизации
производительности и улучшении отзывчивости системы.

** STARTTIME: Информация о времени запуска процесса может быть
полезна для отслеживания длительности работы процесса или для
синхронизации событий в системе. Пользователь может использовать время
запуска для определения, сколько времени прошло с запуска процесса или для
сравнения времени запуска различных процессов. Это может быть полезно,
например, для определения, как долго процесс выполняется или для анализа
зависимостей между процессами.

** COMMAND: Информация о команде запуска процесса может быть
использована для отслеживания и анализа выполняющихся команд.
Пользователь может изучать команды, с которыми запущены процессы, чтобы
понять, какие задачи и операции выполняются в системе. Это может быть
полезно для мониторинга и отладки процессов, а также для идентификации
специфических команд, вызывающих проблемы или потребляющих много
ресурсов.

Ниже приведена информация выводимая при нажатии на кнопку
«Информация о процессоре».

** Model name: Этот параметр содержит информацию о модели
процессора. Он указывает на конкретную модель и характеристики
процессора, такие как производитель, частота и количество ядер.
Пользователь может использовать эту информацию для идентификации
процессора и определения его характеристик для анализа производительности
или совместимости с определенными программами или операционными
системами.

** Vendor ID: Этот параметр указывает на производителя процессора. Он
предоставляет информацию о компании, выпустившей процессор.
Пользователь может использовать эту информацию для идентификации
производителя процессора и выбора определенных драйверов или
оптимизаций, связанных с производителем.

** CPU Family: Этот параметр указывает на семейство процессора. Он
обозначает общую архитектуру или поколение процессора. Пользователь
может использовать эту информацию для определения совместимости
программного обеспечения с определенным семейством процессоров или для
анализа различий между разными поколениями процессоров.

** CPU Cores: Этот параметр указывает на количество ядер процессора. Он
определяет, сколько независимых вычислительных ядер имеет процессор.
Пользователь может использовать эту информацию для определения
параллельных возможностей процессора и распределения задач между ядрами
для повышения производительности.

** Microcode: Этот параметр содержит информацию о микрокоде
процессора. Микрокод – это набор инструкций, используемых процессором
для выполнения определенных операций. Пользователь может использовать
эту информацию для обновления микрокода процессора и улучшения его
стабильности или исправления известных ошибок.

** Cache Size: Этот параметр указывает на размер кеша процессора. Кеш –
это быстрая память, используемая процессором для временного хранения
данных, к которым он обращается часто. Размер кеша влияет на скорость
доступа к данным и может существенно повлиять на производительность
процессора. Пользователь может использовать эту информацию для анализа и
сравнения производительности различных процессоров.

** Bogomips: Этот параметр указывает на скорость исполнения процессора,
измеряемую в богомипах (bogomips). Богомипс – это синтетическая мера
производительности процессора, которая не отражает реальную
производительность, а используется в основном для сравнения разных
процессоров. Пользователь может использовать эту информацию для оценки
относительной производительности разных процессоров.

** Address Sizes: Этот параметр указывает на размер адресов,
используемых процессором. Он определяет максимальное количество памяти,
которую процессор может адресовать. Пользователь может использовать эту
информацию для оценки доступной памяти и совместимости с приложениями
или операционными системами, требующими определенного размера
адресного пространства.

Ниже приведена информация выводимая при нажатии на кнопку
«Общая информация о системе».

** Operating system: Выводится название операционной системы и ее
версия. В данном случае операционная система - Linux 6.5.0-28-generic. Эта
информация полезна для идентификации операционной системы и ее версии,
что может быть важно при установке или настройке программного
обеспечения.

** CPU: Выводится информация о модели процессора. В данном случае
используется процессор Intel(R) Core(TM) i5-9300H CPU @ 2.48GHz. Эта
информация указывает на модель и тактовую частоту процессора.
Температура процессора (CPU Temperature): Выводится текущая
температура процессора в градусах Цельсия. В данном случае температура
процессора составляет 43,9 градусов Цельсия.

** Machine type: Выводится тип машины или архитектуры процессора. В
данном случае тип машины указан как x86_64, что обозначает 64-битную
архитектуру процессора.

** User ID: Выводится идентификатор текущего пользователя. В данном
случае идентификатор пользователя равен 1000.

** Group ID: Выводится идентификатор текущей группы пользователя. В
данном случае идентификатор группы равен 1000.

** Effective user ID: Выводится эффективный идентификатор текущего
пользователя. В данном случае эффективный идентификатор пользователя
равен 1008.

** Effective group ID: Выводится эффективный идентификатор текущей
группы пользователя. В данном случае эффективный идентификатор группы
равен 1000.

** Number of processors: Выводится количество процессоров в системе. В
данном случае количество процессоров равно 8.

** Total RAM: Выводится общий объем оперативной памяти в мегабайтах.
В данном случае общий объем оперативной памяти составляет 7737 МБ.

** Free RAM: Выводится объем свободной оперативной памяти в
мегабайтах. В данном случае объем свободной оперативной памяти составляет
1891 МБ.

Ниже приведена информация выводимая при нажатии на кнопку
«Информация о подключённых устройствах».

** Bus: Это номер шины USB, к которой подключено устройство. Шина USB
представляет собой физический интерфейс, который обеспечивает связь между
компьютером и устройством.

** Device: Это номер устройства на шине USB. Каждое устройство,
подключенное к шине USB, имеет свой уникальный номер.

** ID: Это идентификаторы производителя и продукта устройства. Он
состоит из двух значений, разделенных двоеточием. Первое значение
(идентификатор производителя) указывает на уникальный код производителя
устройства. Второе значение (идентификатор продукта) указывает на
уникальный код самого устройства.

** Device: Это описание устройства. В данном случае, описание может
включать имя или описание устройства. Например, «xHCI Host Controller»
описывает контроллер хоста USB, «Unknown» указывает на то, что описание
устройства неизвестно, а «HP Wide Vision HD Camera» описывает камеру HP
Wide Vision HD.

Ниже приведена информация выводимая при нажатии на кнопку
«Информация о жёстких дисках».

** NAME: Это имя устройства. Оно может быть связано с именем
устройства типа «loop» или «sd».

** RM: Это флаг, указывающий, является ли устройство съёмным
(removable) или нет. Значение «1» обычно означает, что устройство съёмное, а
«0» - что нет.

** SIZE: Это размер устройства в мегабайтах. Размер рассчитывается на
основе значения, полученного из файла «/sys/block/<device_name>/size».
Значение размера измеряется в блоках, и затем конвертируется в мегабайты.

** RO: Это флаг, указывающий, доступно ли устройство только для чтения
(read-only) или нет. Значение «1» обычно означает, что устройство доступно
только для чтения, а «0» - что доступно и для чтения, и для записи.

** MOUNTPOINTS: Это точки монтирования устройства. Они получаются
из файла «/proc/mounts». Каждая точка монтирования указывает на путь, по
которому устройство примонтировано в файловой системе.

Ниже приведена информация выводимая при нажатии на кнопку
«Информация об ядре».

** Operating System: Это переменная содержит название операционной
системы, на которой выполняется код. Она указывает на конкретную
операционную систему, такую как Linux, Windows, macOS и т. д.

** Node Name: Это переменная содержит имя узла или компьютера в сети.
Она представляет собой идентификатор, который может быть присвоен
компьютеру в сети. Имя узла обычно используется для идентификации
компьютера в локальной сети.

** Kernel Release: Это переменная содержит версию ядра операционной
системы. Версия ядра определяет конкретную версию программного
обеспечения ядра операционной системы. Она может включать номер версии и
дополнительные идентификаторы, которые указывают на конкретный релиз
ядра.

** Kernel Version: Это переменная содержит подробную информацию о
версии ядра операционной системы. Обычно она включает дополнительные
сведения, такие как дату сборки и другие дополнительные параметры. Она
предоставляет более полную информацию о версии ядра, чем просто номер
релиза.

** Machine Hardware Name: Это переменная содержит название аппаратной
платформы или архитектуры компьютера. Она указывает на тип и модель
аппаратного оборудования, на котором работает операционная система. 
Например, значение переменной может быть «x86_64», что означает, что
используется 64-битная архитектура.

Ниже приведена информация выводимая при нажатии на кнопку
«Информация о модулях установленных на ядро».

** Module name: Это имя загруженного модуля ядра.

** Size: Это размер модуля в памяти.

** Used: Это информация о том, какие процессы или другие модули
используют данный модуль.

Ниже приведена информация выводимая при нажатии на кнопку
«Информация о сетевых соединениях».

** Netid: Это идентификатор сетевого протокола, используемого для
соединения.

** State: Это состояние соединения. Для TCP соединений, возможные
значения состояния включают «LISTEN» (ожидание соединения),
«ESTABLISHED» (установленное соединение), «CLOSED» (соединение
закрыто) и т.д. Для UDP соединений, обычно используется значение
«UNCONNECTED» (несоединенное состояние).

** Recv-Q: Это размер очереди приема (receive queue), то есть количество
байтов, ожидающих приема для данного соединения.

** Send-Q: Это размер очереди отправки (send queue), то есть количество
байтов, ожидающих отправки для данного соединения.

** Local Address: Port: Это адрес и порт на локальной машине, связанные с
соединением. Адрес может быть как IPv4, так и IPv6, а порт - числовым
значением, указывающим конечную точку соединения.

** Peer Address: Port: Это адрес и порт удаленной машины, с которой
установлено соединение. Адрес может быть как IPv4, так и IPv6, а порт -
числовым значением, указывающим конечную точку соединения.

** Process : Это информация о процессе, связанном с данной сетевой связью.
В выводе указаны пользователь и идентификатор процесса (PID), а также
другие дополнительные сведения, такие как файловые дескрипторы (fd).
